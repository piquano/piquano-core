2. Haeufige Aufgaben

Service neu starten

sudo systemctl restart crm.service          # CRM
sudo systemctl restart ats-prod.service      # ATS
sudo systemctl restart piquano-app.service   # App
sudo systemctl restart lms-prod.service      # LMS
sudo systemctl restart support-prod.service  # Support

Service-Status pruefen

systemctl status crm.service
sudo journalctl -u crm.service -n 50 --no-pager

Code deployen

cd /opt/<app>
git pull origin prod    # (oder main bei LMS/Support)
sudo systemctl restart <service>.service

3. Backups

Taeglich um 03:00 UTC: 14 Datenbanken + Media + Configs, GPG-verschluesselt auf Hetzner Storage Box.

# Backup-Log pruefen
tail -20 /var/log/piquano/pg-backup-$(date +%Y-%m-%d).log

# Restore
/opt/piquano-backup/restore.sh list                  # Backups anzeigen
/opt/piquano-backup/restore.sh db piquano_crm         # DB restoren
/opt/piquano-backup/restore.sh full                    # Gesamtes System

App nicht erreichbar? systemctl status <service> → journalctl -u <service> -n 50 → systemctl restart <service>

VPS komplett weg? SSH-Key + GPG-Key aus 1Password → /opt/piquano-backup/restore.sh full → DNS umstellen.
